Structured Query Language Injection

SQL injection is a type of cyber attack that targets the vulnerabilities in a website's database layer. It occurs when an attacker inserts malicious SQL code into input fields, such as login forms or search boxes, which are then executed by the website's database.
The goal of SQL injection attacks is typically to extract sensitive information from the database, modify or delete data, or execute administrative tasks on the database server. This can lead to data breaches, loss of sensitive information, and even complete system compromise.

Causes

Lack of Input Validation: Web applications often accept user inputs through forms, URLs, or cookies. When these inputs are not properly validated or sanitized, attackers can inject malicious SQL code.

Dynamic SQL Queries: When web applications construct SQL queries dynamically by concatenating user input with the SQL query string without proper sanitization, it creates a vulnerability. This is common in applications using string concatenation or interpolation to build SQL queries.

Insufficient Error Handling: Error messages returned by the database can reveal information about the structure of the database or the SQL query being executed. Attackers can exploit poorly handled errors to deduce the structure of the database and craft malicious SQL queries accordingly.

In SQL injection is a prevalent vulnerabilities that attackers to exploit to gain unauthorized access to databases & SQL injection manifests in web attacks:-

Data Extraction: Attacker injects the malicious SQL code into input parameters in vulnerable web applications this injected code manipulates the SQL query executes in web application to have sensitive data as usernames & passwords , credit card numbers in databases.
Data Manipulation: Used to modify/delete data within the databases by injecting the command such as “INSERT, UPDATE, DELETE” can alter contents in tables, potentially data loss.
Authentication Bypass: It can exploit to bypass authentication that circumvents login checks that allows them to access restrict areas of application.

Recommendations
Use parameterized queries: instead of dynamically constructing SQL queries by concatenating the user input, that parameterized the queries by database framework. these method separates the SQL code from user input, preventing the attackers from injecting the mailicous SQL commands.

Principle of least astonishment: design the database schema and application logic to minimize the unexpected behavior and limit the potential SQL injection vulnerabilities by use clear naming conventions and consistent coding practices to reduce the likelihood of inadvertently introducing the vulnerabilities.
